# Roadmap Deep Dive – IDApTIK

This document expands on the [main Roadmap](3-Roadmap) with technical implementation details.

## Phase 1: Core System Foundations (Current)

### Completed
- Basic network simulation (star topology, subnets, routing)
- Device types (laptops, servers, routers, cameras, terminals)
- Terminal with filesystem navigation and basic commands
- SSH between devices with device state management
- DNS resolution via configurable DNS servers
- Traceroute showing hop paths
- Zone-based layout (LAN, VLAN, External)

### In Progress
- Defining hacker view experience (physical-first access, grounded gameplay)
- Device interface design (basic GUI for servers/routers, fuller GUI for personal computers)
- Network discovery mechanics (progressive reveal vs immediate visibility)

### Next Steps
- Implement "grounded" hacker start (begin from a specific device, not god-mode)
- Basic Uplink-style GUI for server/router interaction
- Local network view from device perspective (only see what's reachable)
- InterNIC concept for public server directory

## Phase 2: Network Depth & Security

### Trace System
- Active trace timer during intrusions
- Bouncing through machines to slow trace
- Action-based alerts (failed logins, port scans)
- Passive logging for post-incident investigation

### Firewall Implementation
- Per-subnet firewall rules
- Block/allow by port, IP, protocol
- Firewall rule discovery and exploitation
- Misconfiguration as gameplay opportunity

### Pivoting Mechanics
- SSH session chaining
- Attack from compromised machine's perspective
- Network view updates based on current position
- Physical access bridging (infiltrator plugs in device)

## Phase 3: Environmental Integration

### Physical World Effects
- Power grid simulation (blackouts affect devices and player)
- Network infrastructure damage (cut cables, destroyed equipment)
- Signal strength for wireless connections
- Location-based risk (hacking from inside target building vs remotely)

### Infiltrator Integration
- Camera feed access/manipulation
- Door and access control systems
- Security system status visible to hacker
- Real-time coordination mechanics

### Adaptive Security AI
- Security responds to repeated tactics
- Anomaly detection based on player behavior
- Countermeasure escalation
- Guard dispatch based on alerts

## Phase 4: Mission Framework

### Mission Types
- Data extraction (find and exfiltrate files)
- System sabotage (destroy, corrupt, disable)
- Backdoor installation (persistent access for future missions)
- Support operations (camera feeds, door access for infiltrator)
- Evidence manipulation (plant or destroy)
- Cover-up operations (delete logs, remove traces)

### Persistent World Elements
- Backdoors installed in previous missions remain accessible
- Reputation/heat system based on past actions
- Recurring locations with memory of previous intrusions
- Consequences that carry forward

## Technical Architecture Notes

### Current Stack
- **Frontend**: PixiJS + ReScript (compiled to JS)
- **Build**: Vite
- **State**: ReScript records with mutable fields where needed

### Planned Additions
- Mission definition format (JSON or custom DSL)
- Save/load system for mission progress
- Network state serialization
- Multiplayer synchronization layer (future)
